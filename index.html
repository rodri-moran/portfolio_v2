02 — Proyectos

Proyectos destacados

🏙️

Civix

Reportes urbanos para municipios · Backend distribuido con microservicios

Plataforma pensada para que ciudadanos reporten problemas urbanos a su municipio, como baches, luminarias o incidencias en la vía pública. El proyecto fue una oportunidad para trabajar una arquitectura basada en microservicios, con servicios desacoplados, bases de datos separadas y una estructura preparada para crecer.

Decisiones de arquitectura

API Gateway centraliza autenticación y enrutamiento Auth Service independiente con JWT — no acoplado al negocio User Service y Report Service con bases de datos separadas (PostgreSQL por servicio) Docker Compose orquesta el ambiente completo con un solo comando Contratos via DTOs — ningún servicio expone entidades internas
Java 17 Spring Boot Spring Security JWT PostgreSQL Docker Docker Compose API Gateway REST
🏠

CasaAgenda

Sistema de gestión de alquileres temporales · Angular + Spring Boot

Aplicación para gestionar propiedades y alquileres temporales desde un solo lugar: permite registrar inmuebles, administrar reservas, manejar imágenes y centralizar la información del negocio. El backend está desarrollado en Spring Boot con APIs REST y arquitectura en capas, mientras que el frontend en Angular permite una interacción fluida y desacoplada del dominio.

Stack técnico

Spring Boot con arquitectura por capas (Controller / Service / Repository) DTOs como contratos entre capas — sin exponer entidades JPA Angular con Signals y standalone components (no NgModules) Subida y manejo de imágenes con almacenamiento persistente
Java Spring Boot Angular 17 Signals PostgreSQL DTOs Bootstrap
⚖️

Decision Platform

Reglas de negocio versionadas · Arquitectura desacoplada para decisiones dinámicas

Plataforma diseñada para gestionar y ejecutar reglas de negocio de forma desacoplada. La idea es permitir que decisiones como validaciones, scoring o evaluaciones de fraude puedan evolucionar con versionado, trazabilidad y una arquitectura preparada para sistemas distribuidos.

Decisiones de arquitectura

Rule Service y Decision Service desacoplados — cada uno con su responsabilidad Audit Service independiente con traceId para trazabilidad end-to-end Versionado de reglas — cambios sin afectar decisiones históricas Explainability: cada decisión devuelve las reglas que la determinaron
Java Spring Boot Microservices PostgreSQL Redis Kafka Keycloak Docker

04 — Sobre mí

Quién soy

Soy egresado de la Tecnicatura Universitaria en Programación (UTN Córdoba) y vengo construyendo proyectos donde priorizo no solo que algo funcione, sino también cómo está diseñado por dentro.

Me interesa especialmente el backend porque disfruto pensar la estructura de los sistemas: cómo circulan los datos, cómo se definen los contratos entre servicios y qué decisiones técnicas hacen que una aplicación sea más clara, mantenible y escalable.

Actualmente también estoy comenzando un curso anual de inglés en la Facultad de Lenguas (UNC) para mejorar mi nivel y poder trabajar en entornos técnicos donde el idioma es clave.

Estoy buscando mi primera oportunidad profesional en IT para seguir creciendo dentro de un equipo, aprender de entornos reales de desarrollo y aportar desde el primer día con una base sólida en Java, Spring Boot, SQL y diseño de APIs.
